Introduction

About

This is a general purpose Discord bot designed for all types of communities.

Features

Slash Commands

  • apod

    Sends the Astronomy Picture of the Day.

  • dadjoke

    Sends a random dad joke.

  • numberfact

    Get a random fact about a number.

    Parameters: number - The number to get a fact about.

  • poll

    Create a poll.

    Parameters: normal - Create a normal yes or no poll.
    straw - Create a strawpoll.
    ranked - Create a ranked poll.

  • qrcode

    Generates a QR code from the given data.

    Parameters: data - The data to convert to a QR code.

  • thispersondoesnotexist

    Sends a random image of a person that does not exist.

  • botinfo

    Provides detailed information about the bot.

  • help

    Get a list of all commands.

    Parameters: command - The command to get information on.

  • ban

    Manage guild bans.

    Parameters: add - Ban a member.
    remove - Unban a member.
    list - List banned members.

  • clear

    Clear up to 99 messages.

    Parameters: amount - Number of messages to clear.

  • kick

    Kick a member.

    Parameters: target - The member to kick.

  • modmail

    Ban a member from using modmail.

    Parameters: ban-add - Ban a member from using modmail.
    ban-remove - Unban a member from using modmail.
    ban-list - List members banned from using modmail.

  • timeout

    Timeout a member.

    Parameters: target - The member to kick.
    time - The time to timeout the member for. Number than units.

  • warn

    Manage warns.

    Parameters: add - Warn a member.
    remove - Remove a warn.
    list - List warnings.

  • avatar

    Get the avatar URL of the selected user, or your own avatar.

    Parameters: target - The user's avatar to show.

  • remind

    Reminds you about something.

    Parameters: list - Lists your reminders.
    remove - Removes a reminder.
    add - Adds a reminder.

  • serverinfo

    Display information about this server.

  • userinfo

    Provides information about the user.

    Parameters: target - The member you want to get information about.

Installation

1. Prerequisites

  • Ensure you have Node.js installed on your system.

  • Make sure to use pnpm as the package manager.

2. Bot Setup

  1. Clone this repository to your local machine or download it as a ZIP archive.

  2. Create a .env file in the project directory. You can use the provided .env.example as a template. Fill in your bot's token and ID. 

    TOKEN=YOU_BOT_TOKEN
CLIENT_ID=YOU_CLIENT_ID
# API KEYS
NASA_API_KEY=YOUR_NASA_API_KEY # Create one at https://api.nasa.gov/ or leave blank to use the demo key.

  3. Install the required dependencies by running the following command in your terminal:

    pnpm install
  4. Initialize the Database 
    node functions/db/init.js

    This script will set up the necessary database tables. If you ever make changes to the database models, you can use the --force or -f option to force synchronization of the tables. Be cautious when using this option as it will empty and recreate the model tables. This will delete all the data you previously had stored.

3. Deploy and Run

  • To deploy or reset the bot's commands, run the following command:

    node deploy-commands.js

  • To start the bot, run:

    node index.js

4. Invite the Bot to Your Server

Creating a Discord Bot Application

  1. Visit the Discord Developer Portal.

  2. Click on the "New Application" button.

  3. Give your application a name.

  4. Navigate to the "Bot" section in the left sidebar.

  5. Scroll down to the "Privileged Gateway Intents" section.

  6. Enable all the intents under this section. These intents are required for various features of the bot. To enable these intents, simply toggle the switches to the "ON" position.

  7. Save your changes.

Inviting the Bot

Invite the bot to your server by using the following link, replacing <YOUR_CLIENT_ID> with your bot's client ID: 

https://discord.com/api/oauth2/authorize?client_id=<YOUR_CLIENT_ID>&permissions=8&scope=bot%20applications.commands

Once the bot is in your server, you can start using its features and commands.

Configuration

Configure the bot on the settings page. Enable or disable features using the checkboxes next to each feature. Lack of necessary information will disable a feature. Note that improper configuration may break the bot, e.g., using non-existent emojis in the Starboard Emojis section.

Replacements

For the text inputs that control messages the bot will send, you can use the following replacements.

Text to Type Replaced with Features
{client.mention} Mentions the bot All
{client.username} Bot's username All
{client.id} Bot's user ID All
{client.tag} Bot's tag (username#discriminator) All
{user.mention} Mentions the user that the feature is related to. Welcome Message, Leave Message, Message Functions
{user.username} The username of the user that the feature is related to. Welcome Message, Leave Message, Message Functions
{user.id} The ID of the user that the feature is related to. Welcome Message, Leave Message, Message Functions
{user.tag} The tag (username#discriminator) of the user that the feature is related to. Welcome Message, Leave Message, Message Functions
{channel.mention} Mentions the channel the feature is related to. Message Functions
{channel.name} The channel's name the feature is related to. Message Functions
{channel.id} The channel's ID the feature is related to. Message Functions
{channel.topic} Channel's topic the feature is related to. Message Functions
{guild.name} Server (guild) name Welcome Message, Leave Message, Message Functions
{guild.id} Server (guild) ID Welcome Message, Leave Message, Message Functions
{guild.memberCount} Member count in the server Welcome Message, Leave Message, Message Functions
{message.content} Content of the user's message Message Functions
{message.id} ID of the user's message Message Functions
